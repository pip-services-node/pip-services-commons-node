Interface ICommandInterceptor

Interface for stackable command intercepters, which can modify the message execution pipeline. Command interceptors are used to intercept calls, perform a set of actions, and, optionally, cancel the command's actual execution by simply returning a result.

A command’s return value can also be intercepted in a similar manner. For example: the result can be written to cache, so that the next call doesn’t have to be made.

This mechanism can be used for authentication, logging, and other functions.

Command interceptors are also often used for intercepting a command's unique call and replace it with a universal "message transfer" call.
